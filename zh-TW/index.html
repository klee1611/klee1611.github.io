Stateless HTTP, Stateful Session and Cookies

發表於 2021-06-28 分類於 Web Disqus：

Stateless HTTP

HTTP 是一種 stateless 的 protocol，
也就是說每一次的 request / response 都是獨立的，
和之前或之後的 request / response 無關。
相同的 request 就會回應相同的 response，
不會因為之前的 request / response 內容而有不同。

這樣一來 server 因為不需要儲存使用者資訊可以省去大量的資料庫、伺服器儲存空間，
也因為不需要讓 client 每次都必須連線到相同的 socket 而能夠加快 response time 和省去不少 network bandwidth，

WSL 2 和 WSL 1 的差異

WSL 2 是基於 Hyper-V 在 virtual machine 中跑完整的 Linux kernal，
WSL 1 則是在 Windows 系統上對 Linux 功能的模擬，
因此 WSL 2 比 WSL 1 支援更多 Linux 原生的功能和 system call。

  • Process group 是一堆 related processes 組成的集合

Process Group

一堆 related processes 共享相同的 process group identifier (PGID)。
這群 process 裡面會有一個 process group leader 是建立這個 process group 的 process，
這個 process group leader 的 PID 就會是這個 process group 的 PGID。
